# 探索地图路线汇合设计

## 背景

当前探索地图由 `src/server/game/map.ts` 生成，地图固定为 10 层，每层 2 到 4 个节点。节点本身已经有稳定的 `x` 坐标，前端按这些坐标绘制节点和连线。

现有问题集中在 `connectMapLayers()` 的连线策略：它先让每个上层节点连接最近的下层节点，然后为了保证下层节点都可达，会对没有入边的节点补边。这个补边规则能保证可达性，但在节点分布较密或下一层节点较多时，容易产生多条线反复交叉。玩家看到的效果不是“几条路线在某处汇合”，而是“很多路线互相穿插”。

用户期望的地图路线更接近：部分情况下几条路汇到一起；少数情况下所有路线都汇到同一个点，然后再度分开。路线应保留随机探索感，但红框区域不应出现多条路多重交叉。

## 目标

- 保留现有地图数据结构，不新增节点字段。
- 保留 10 层、每层 2 到 4 个节点、每个非末层节点至少 1 条出边、最多 2 条出边的基础约束。
- 让“局部汇合”成为常见拓扑：相邻或距离近的 2 条路线可以汇到同一个下层节点，少量情况下允许 3 条路线汇合。
- 让“全路汇合再分开”成为低频拓扑：某些层当前所有可前进路线会汇到下一层同一个节点，并在后续层重新分叉。
- 降低补边带来的视觉交叉，补边只服务于可达性，不主动制造复杂交叉。
- 保持所有完整路径从入口可达末层，且玩家战数量仍在现有安全范围内。
- 保持第一张地图无休息点的节奏约束。

## 非目标

- 不重做探索地图 UI。
- 不新增节点类型、奖励类型、装备、遗物或图标资源。
- 不修改战斗、商店、事件、休息点的结算逻辑。
- 不修改装备数值、遗物数值、战斗参数或外部数值 Excel。
- 不引入路线生成库或独立图布局系统。

## 推荐方案

采用“最小连线策略修正”：只调整 `connectMapLayers()` 及其私有辅助函数，不改公开类型和地图存档结构。

生成每一层连线时，先按当前层和下一层节点的横向位置建立主结构，再根据本层的汇合模式决定目标分配：

- 普通层：优先连接最近的下层节点，允许相邻上层节点自然汇到同一个目标。
- 局部汇合层：选择一个合适的目标节点，让 2 条或 3 条距离接近的上层路线汇入该目标。
- 全路汇合层：选择一个瓶颈目标，让当前层所有上层节点都汇入该目标。该目标后续仍按普通规则继续连出，从而形成“汇合后再分开”的节奏。

连线完成后，再执行最小补边：

- 只处理完全没有入边的下层节点。
- 优先从不会与已有边交叉的上层节点补一条边。
- 如果无法完全避免交叉，则选择交叉数量最少、横向距离最近、且出边数未超过 2 的来源。
- 不为了“每个节点看起来都有多条路”额外补边。

## 汇合目标选择

局部汇合和全路汇合都需要稳定、可复现的目标选择。目标选择使用现有 `createRng()`，以 `mapIndex`、层号、当前层节点数、下一层节点数等信息组成 seed，保证同一 run 的地图稳定。

目标优先级：

1. 下一层存在 `PLAYER_BATTLE` 时，优先选择靠近中间或靠近参与汇合路线的玩家战节点。
2. 没有合适玩家战节点时，选择靠近下一层中线的节点。
3. 避免选择极端边缘节点作为全路汇合点，除非下一层只有 2 个节点且随机结果明确落在边缘。

局部汇合的来源选择：

- 按目标节点的横向距离排序。
- 选择距离目标最近的 2 个上层节点。
- 在节点数足够且随机命中时，扩展到 3 个上层节点。

全路汇合的来源选择：

- 当前层所有节点都连接到同一个目标。
- 全路汇合层优先保证“所有可走路线汇到同一个点”的视觉意图；该目标所在层的其他节点不强制从上一层补入边。
- 只有当补边不破坏“全路汇到同一点”的视觉意图、且不会产生明显交叉时，才允许给该目标所在层的其他节点补入边。
- 无论是否补入边，都必须保证至少一条完整路线从入口走到末层。

## 频率控制

汇合应是地图节奏的一部分，不应每层都发生。

建议频率：

- 局部汇合：多数地图可出现 1 到 3 次。
- 全路汇合：低频出现，约每张地图 0 到 1 次；跨多个 seed 必须能观察到。
- 全路汇合不放在第 0 层到第 1 层之间，避免开局选择立即失效。
- 全路汇合不放在最后一层之前，避免汇合后没有重新分叉的空间。

实现上可以先以确定性随机决定某一层是否为全路汇合层，再在剩余层中按普通或局部汇合规则生成。

## 交叉控制

把每条边视为从当前层节点 `x` 到下一层节点 `x` 的线段。相邻层内两条边如果来源顺序和目标顺序相反，则视为交叉。

补边时计算候选来源与已有边的交叉数量，排序规则为：

1. 交叉数量更少。
2. 来源节点当前出边数更少。
3. 来源与目标横向距离更近。

这样可以保留必要可达性，同时避免截图中红框区域那种多条线互相穿过的观感。

## 测试计划

先补失败测试，再实现代码。

新增或更新 `src/server/game/map.test.ts`：

- 多个固定 seed 中，应能观察到玩家战或普通节点拥有 2 到 3 条入边，证明局部汇合存在。
- 多个固定 seed 中，应能观察到至少一张地图某一层的所有当前层节点都连接到下一层同一个节点，证明全路汇合存在。
- 全路汇合节点之后必须继续分叉：该节点应连接到 2 个不同的后续节点，或在下一层内形成至少 2 条不同的完整后续路径。
- 每个非末层节点出边数量仍为 1 到 2。
- 所有出边只连接下一层。
- 所有完整路径仍能从入口抵达末层。
- 完整路径上的 `PLAYER_BATTLE` 数量仍在 3 到 5 之间，沿用当前测试的安全范围。
- 第一张地图仍不生成 `REST`。

实现后运行：

```bash
npx vitest run src/server/game/map.test.ts
npm run build
```
