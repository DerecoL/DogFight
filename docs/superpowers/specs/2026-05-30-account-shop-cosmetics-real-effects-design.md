# 账号商城外观真实生效设计

## 目标

账号商城中售卖的称号、头像、主页背景、狗狗皮肤和战斗特效，购买并装备后必须在游戏界面中产生可见效果；这些效果只改变展示，不改变战斗数值。

## 范围

- 称号：显示在账号展示区域，玩家能明确看到当前装备的称号。
- 头像：显示在账号展示区域，区分骨头头像和皇冠头像。
- 主页背景：装备后改变大厅和账号相关界面的背景风格。
- 狗狗皮肤：装备后改变玩家侧狗狗展示，先覆盖商城已有的柴犬围巾和雪原萨摩耶。
- 战斗特效：装备后改变玩家侧投骰与战斗播放的视觉风格，先覆盖金色骰光和极光投掷。

## 架构

根组件维护一份 `CosmeticsResponse`，登录、购买、装备后刷新。`Shell` 根据已装备外观生成展示资料和 CSS class，传给顶部栏、背景层和战斗视图。狗狗皮肤通过 `DogBadge` 的数据属性和 CSS 叠加表现，不新增大图资源，避免影响加载速度。

## 测试

新增前端结构测试验证：

- 根组件加载并刷新 `/cosmetics/me`。
- `Shell`、`TopBar`、`BattleView` 和 `DogBadge` 接收外观状态。
- CSS 存在背景、头像、皮肤、战斗特效的可见样式。

构建前运行相关测试，最终运行 `npm run build` 同步单文件版本。
