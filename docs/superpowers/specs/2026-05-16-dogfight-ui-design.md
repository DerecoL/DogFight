# DogFight 游戏 UI 设计规格

日期：2026-05-16

## 目标

为 DogFight 设计一套完整的 Figma UI 方向，覆盖完整可玩流程：

1. 登录和注册
2. 选择狗狗
3. 选择商店类型
4. 商店工作台
5. 匹配确认
6. 战斗回放和结算

整体设计需要轻松、活泼、有辨识度，采用手绘绘本桌游风格。PC 网页是主要目标。移动端只要求可用，并在设计文档中说明适配原则，不做完整移动端高保真设计。

## 产品背景

DogFight 是一个 React/Vite 网页游戏，后端使用 Fastify 和 Prisma。当前核心循环如下：

1. 选择一个狗狗类型。
2. 进入商店阶段。
3. 购买、出售、刷新商品，并在装备栏和背包格子中摆放物品。
4. 匹配一个对手。
5. 观看由骰子点数和物品触发驱动的自动战斗回放。
6. 继续本局，或在结算后重新开始。

UI 必须支持当前已有机制：

- 每种狗狗有独立被动特性。
- 不同商店类型会影响商品池。
- 装备栏为 15 格单行，并按从左到右的顺序触发。
- 背包为 2 x 10 格，用于存放战斗中默认不生效的物品。
- 物品包含尺寸、触发骰点、价格、折扣、伤害或治疗效果。
- 战斗回放包含行动方、骰点、物品触发、生命变化、日志和结果。

## 已选方向

采用“绘本章节式流程”，而不是把所有功能压进一个复杂面板。

游戏整体应像一本摊在桌上的手绘桌游书。每个主要阶段都是本局中的一个章节页面：

登录 -> 狗狗图鉴 -> 商店选择 -> 商店工作台 -> 匹配确认 -> 战斗回放 / 结算

共享外壳通过顶部状态条和章节进度保持连续性。每个章节的主内容区根据当前阶段单独组织。

## 视觉风格

已选择的视觉方向是“绘本桌游”。

核心特征：

- 页面 UI 间需要保持统一的设计元素。
- 温暖的纸张底色。
- 深棕色粗手绘描边。
- 略不规则的圆角。
- 轻微错位的纸片或卡牌阴影。
- 明亮但偏温暖的强调色。
- 资源、状态和提示使用贴纸式徽章。
- 战斗特效使用手绘粒子、纸屑、星星和冲击线。

建议色彩：

- 纸张背景：奶油色 / 暖白色。
- 描边和正文：深棕色。
- 主操作：温暖蓝色。
- 品牌强调：橘红色。
- 货币和奖励：黄色。
- 治疗和安全状态：薄荷绿。
- 伤害和危险提示：粉红红色。
- 通用装备辅助色：天蓝色和淡紫色。

字体规则：

- 所有功能性中文文本使用高可读性的中文无衬线字体。
- Logo、章节标题和大型按钮可以使用圆润或手绘感的展示字形。
- 密集玩法信息不能为了手写感牺牲可读性。

Figma 文件建议按以下页面或分区组织。

### 01 Foundations

定义：

- 颜色 token
- 字体层级
- 手绘描边规则
- 阴影和层级规则
- 圆角规则
- 间距尺度
- 图标和徽章风格

### 02 Components

创建可复用组件：

- 主按钮、次按钮、危险按钮、图标按钮
- 文本输入框
- 顶部状态条
- 章节进度条
- 资源徽章
- 狗狗卡片
- 狗狗详情面板
- 商店类型卡
- 商店物品卡
- 物品详情面板
- 装备格和背包格
- 已放置物品块
- 出售拖放区
- 战斗狗狗面板
- 生命条
- 骰子显示
- 战斗日志行
- 结果卡

关键状态：

- 默认
- 悬停
- 选中
- 禁用
- 金币不足
- 拖拽中
- 可放置目标
- 不可放置目标
- 物品触发高亮
- 胜利
- 失败

### 03 Screens - PC

创建 PC 高保真画板，建议尺寸为 1440 x 900：

1. 登录 / 注册
2. 狗狗图鉴
3. 商店选择
4. 商店工作台
5. 匹配确认
6. 战斗回放
7. 结算状态，可以作为独立画板，也可以作为战斗画板的状态变体

### 04 Responsive Notes

记录移动端适配原则，但不产出完整移动端高保真设计：

- 顶部状态条在窄屏下纵向堆叠。
- 多栏布局改为纵向分区。
- 装备栏和背包格保持横向滚动。
- 关键主操作在需要时靠近底部并保持易触达。
- 保留物品名称和骰点触发信息的可读性。

## 屏幕设计

### 登录 / 注册

登录页应像一本绘本的封面。

布局：

- 中心放置纸张质感表单卡。
- 顶部展示游戏标识和标题。
- 用一句简短副标题说明核心循环。
- 提供邮箱和密码输入框。
- 提供登录和注册按钮。

背景可以有轻量桌面道具，但表单必须是最清晰、最主要的信息层。

### 狗狗图鉴

狗狗选择页必须支持未来扩展，不应固定为三张卡片。

布局：

- 主体为可滚动的狗狗卡片网格。
- 搭配一个当前选中狗狗的详情面板。
- 提供明确的“开始一局”主操作。

要求：

- 狗狗卡片网格应能自然容纳更多狗狗类型，不需要重画页面。
- PC 一屏建议展示约 6-8 张狗狗卡片，更多卡片通过滚动访问。
- 每张卡片展示手绘头像、名字、特性摘要和标签。
- 详情面板展示更大的角色图、被动特性、策略说明、难度或标签，以及开始按钮。
- 预留未来筛选或分类标签位置，例如“稳定”“爆发”“随机”“新手推荐”。

### 商店选择

该屏幕用于让玩家选择下一步进入哪种商店。

布局：

- 六种商店类型做成手绘招牌或小摊位卡片。
- 每张卡说明该商店偏向的物品类型。
- 选中卡片需要有明确的抬起和描边状态。
- 选择后主继续按钮保持突出。

该页面应帮助玩家理解不同商店的差异，而不是堆长段规则文字。

### 商店工作台

这是信息密度最高、也是最重要的核心玩法页面。

布局：

- 左侧或上方：当前商店商品和刷新操作。
- 中间：装备栏和背包格。
- 右侧：选中物品详情面板。
- 商店阶段拖拽时，出售拖放区保持可见。
- 玩家准备好后可以进入匹配。

要求：

- 购买、出售、刷新和摆放都要高效。
- 装备栏必须清楚传达从左到右的触发顺序。
- 背包需要明确表达“存储区”，不要让玩家误以为背包物品会在战斗中生效。
- 物品块尺寸稳定，并清晰显示骰点触发信息。
- 选中物品详情面板展示尺寸、骰点、效果、价格、折扣，以及购买或出售操作。
- 拖拽状态需要用手绘高亮清楚表达。

### 匹配确认

该页面在开战前提供一次明确停顿。

布局：

- 对手卡片展示狗狗头像、名字、狗狗类型、胜负数和回合数。
- 展示玩家装备复查区。
- 如果当前逻辑允许，保留最终调整装备的入口或区域。
- 提供明确的“开始战斗”主按钮。

该页面应像翻到下一章时，在对页看到即将对战的对手。

### 战斗回放

战斗回放优先保证清晰，但仍需要足够生动。

布局：

- 顶部为对手装备栏。
- 中间为战斗舞台，包含对手狗狗、骰子显示和玩家狗狗。
- 底部为玩家装备栏。
- 战斗日志默认可折叠。
- 提供播放速度控制。

要求：

- 当前触发的物品必须容易看见。
- 生命变化不能只依赖动画，需要有清晰数字或文本反馈。
- 骰子结果应位于中心且足够醒目。
- 战斗日志可以默认收起，但必须随时可展开。
- 战斗特效使用手绘粒子、星星、斩击线、治疗环和纸片式冲击线。

### 结算状态

结算状态出现在单场战斗后或整局结束时。

布局：

- 在战斗舞台上方或下方展示结果卡。
- 展示胜利、失败或平局结果。
- 展示当前局胜负记录和积分。
- 如果本局继续，提供继续按钮。
- 如果整局结束，提供重新选择狗狗或重新开始按钮。

## 交互说明

导航：

- 章节进度展示当前阶段。
- 登录后顶部状态条保持一致。
- 账号和退出入口保留，但视觉上应弱于玩法信息。

拖拽：

- 拖动物品时，物品像纸片一样抬起。
- 可放置格子使用薄荷绿高亮。
- 拖拽时出售区域变得更活跃。
- 不可放置目标保持克制，不制造过多噪音。

反馈：

- 金币不足时购买按钮禁用，并说明还差多少金币。
- 选中卡片使用粗描边和轻微抬起效果。
- 战斗中触发的物品使用手绘光晕或贴纸爆发效果。

## 可访问性和可用性

- 所有功能性文字必须与纸张背景保持高对比。
- 密集信息区不使用手绘字体。
- 按钮尺寸稳定，标签清晰。
- 关键状态不能只依赖颜色，需要同时使用文字、图标或描边变化。
- 装备和背包格子必须保持固定尺寸，避免布局跳动。

## 不在本次范围

- 完整移动端高保真画板。
- 新游戏机制。
- 新后端或 API 行为。
- 生产代码实现。
- 最终插画资产生产；本次只定义 Figma 风格占位和组件方向。
