# DogFight 手绘美术与反馈表现整体优化设计

## 背景

当前 DogFight 已经形成了奶油色、粗描边、桌游卡牌感的基础视觉方向，但整体仍偏“网页 UI 套手绘色板”：面板、卡片、格子和按钮较规整，细节层次不足，交互反馈和战斗反馈还没有完全体现“手绘桌游”的柔和、精巧和趣味。

本次优化不切换美术风格，不转向暗色霓虹、竞技漫画或硬核街机。目标是在现有风格上做增量精修，让画面更像一款精心制作的手绘桌游，而不是一套方正网页界面。

## 设计目标

1. 保持现有奶油纸面、粗描边、柔和色彩和狗狗主题。
2. 增强手绘感：边缘、描边、贴纸、纸张、桌面、阴影和小装饰都要更自然。
3. 降低机械感：避免所有页面都呈现为死板矩形网格，允许轻微错落、纸片叠放和不规则外观。
4. 提升反馈密度：购买、拖拽、放置、升级、受击、治疗、护盾、中毒、胜负都要有明确视觉因果。
5. 保证可玩性：高频操作区域仍需保持稳定、清晰、易点击，手绘装饰不能干扰信息阅读和拖拽判断。

## 美术方向

### 核心原则：隐形网格，手绘外观

底层布局继续使用稳定网格，保证商店、背包、装备栏和战斗信息不乱。但视觉表层要做成手绘纸片、桌面小物、贴纸、徽章、托盘和小剧场。

具体要求：

- 面板：保留矩形边界用于布局，但外观改成轻微纸片感，可使用伪元素叠出纸边、手绘描边、角落贴纸和柔和投影。
- 描边：不要所有边线都完全等宽、等直。可以通过 SVG 边框、伪元素阴影或多层边框模拟手画线条。
- 背景：从纯色升级为柔和纸面或桌布质感，加入低透明度纹理、压痕、铅笔阴影和桌面暗角。
- 卡片：狗狗、商品、遗物、模式入口都应像摆在桌上的收藏卡或纸片，而不是标准网页卡片。
- 动效：使用轻微晃动、弹回、翻卡、盖章、手写数字、纸屑和星星，不使用强烈霓虹、硬边爆炸或过度粒子。

## 页面级优化

### 模式大厅

模式大厅从普通卡片网格升级为“桌面选择板”：

- 每个模式入口像一张贴在桌上的手绘标签，保留可点击区域但外观可轻微旋转或错落。
- 可用模式使用彩色贴纸、徽章和手绘小图标增强识别；未解锁模式使用胶带、虚线和淡化印章。
- 个人战绩面板做成记录卡或夹在桌面上的成绩单，避免像普通数据面板。

### 狗狗选择

狗狗选择页应更像“收藏卡册”：

- 狗狗卡增加手绘相框、纸张底纹、特性贴纸和底部小标签。
- 选中狗狗时使用柔和跳动、边框亮起、贴纸翻动或小星星，而不是只用硬 outline。
- 详情面板做成大收藏卡，狗狗图像放在柔和纸框或小垫子上，保持现有狗狗图片风格。

### 商店

商店应像“桌面货架”而不是商品列表：

- 商品卡做成小物件摆在纸面或木质托盘上，价格牌改成手写吊牌。
- 购买成功时，商品应有短暂弹起并飞入背包或出现手绘确认贴纸。
- 金币不足时使用轻微左右晃动、价格牌变红、提示贴纸浮出，不只依赖按钮禁用。
- 刷新商店时，商品可以做短翻牌或纸片替换动效。

### 背包与装备栏

背包和装备栏是高频操作区，结构必须稳定，但质感要增强：

- 格子从硬矩形改成布面、纸板或木质托盘凹槽。
- 可放置区域显示柔和绿色手绘描边，非法区域显示轻微红色抖动和叉号草稿线。
- 拖拽物品时增加“手拿纸片”感觉：轻微旋转、阴影拉大、纸片浮起。
- 放下物品时有轻弹落位；出售区吸附时出现撕票或小金币反馈。
- 可升级物品显示贴纸角标；合成升级播放手绘星星、描边闪光和品质边框升阶。

### 战斗页

战斗页是本次优化的最高优先级，需要做成“桌上小剧场”：

- 两只狗不应像放在硬面板里，而是站在柔和舞台、纸板场景或垫子上。
- 骰子区域要成为战斗节奏核心：投骰时有滚动、弹起、落桌阴影和命中提示。
- 装备触发时，高亮应从装备卡传递到目标狗狗，形成“装备触发 -> 飞出效果 -> 命中/治疗/状态变化”的因果链。
- 伤害数字做成手写弹字：普通伤害红色，治疗绿色，护盾蓝色，中毒草绿，虚弱紫色。
- 状态不仅显示芯片，还要在狗狗身上有轻量叠层：中毒毒雾、护盾泡泡、虚弱波纹、冻结霜边、荆棘金刺。
- 胜利/失败应有结果牌翻出、纸屑、狗狗小跳或失落下沉，而不是普通结果块。

### 奖励与遗物

奖励页负责制造惊喜感：

- 职业奖励可使用翻卡、盖章、纸屑或聚光贴纸表现“获得新装备”。
- 遗物选择应像从桌面抽出三枚特殊徽章，强化稀有度、神秘感和收藏感。
- 选择后应有短确认反馈：被选遗物贴到遗物栏，未选项淡出或收回。

## 反馈系统

### 操作反馈

- 点击按钮：纸片轻微下压并回弹。
- 悬浮卡片：轻微抬起、阴影变大、描边更清晰。
- 拖拽中：物品旋转 1-3 度，阴影增强，原位置保留淡影。
- 放置成功：短弹跳和柔和亮边。
- 放置失败：小幅抖动、红色草稿线、简短错误贴纸。
- 购买成功：金币减少有数字跳动，商品飞入目标区域或出现确认贴纸。
- 升级成功：两件物品短暂靠拢，出现星星、闪边和新品质边框。

### 战斗反馈

- 投骰：骰子弹跳落桌，当前行动方轻微高亮。
- 装备触发：触发装备卡闪烁，效果轨迹指向目标。
- 普通伤害：手写红色数字、短受击晃动、狗狗亮白闪一下。
- 治疗：绿色数字上浮、柔和圆环扩散。
- 护盾：蓝色泡泡或描边挡在血条/狗狗前，吸收时显示蓝色碎片。
- 中毒：草绿色毒雾、预损血条、结算时绿色伤害数字。
- 虚弱：紫色下坠波纹，下一次伤害被削弱时显示“减弱”贴纸。
- 冻结：冰蓝霜边、短暂停顿感，不使用过硬的科幻冰效。
- 失效：灰色断线、装备卡轻微熄灭，明确表示该次触发被抵消。

## 资产与样式补充

优先通过 CSS、SVG 和少量 canvas 实现，避免大规模重做资源。

建议新增或整理：

- 手绘纸张纹理 SVG 或 CSS 背景。
- 胶带、贴纸、星星、纸屑、手写爆字等小型 SVG 装饰。
- 状态叠层素材：毒雾、护盾泡泡、虚弱波纹、冻结霜边、荆棘闪光。
- 统一视觉 token：纸面色、桌面色、描边色、柔和阴影、状态色、品质色、动效曲线。
- 手绘特效生成函数：用于 canvas 绘制星星、短线、墨点、气泡和手写数字。

## 技术约束

- 不推翻现有 React 页面结构，优先在 `src/App.tsx` 和 `src/App.css` 的现有组件基础上增量拆分和精修。
- 高频操作区必须保持稳定尺寸，不能因为手绘错落导致拖拽命中区域变化。
- 新增动效需要遵守 `prefers-reduced-motion`，为减少动态效果的用户提供静态或低动效表现。
- 影响游戏行为、前端展示、资源、战斗逻辑或打包逻辑的代码修改后，必须运行 `npm run build`，确认单文件版本 `E:\AI-GPT\DogFight\dist-click\DogFight-standalone.cmd` 同步生成。
- 本设计文档只定义美术与反馈目标，不改变战斗规则、装备数值、狗狗能力、遗物效果或服务器接口。

## 优先级

1. 建立手绘视觉系统：纸面、描边、阴影、按钮、卡片、格子、品质边框。
2. 优化背包和商店：拖拽、购买、出售、升级、刷新反馈。
3. 强化战斗页：骰子、装备触发、伤害/治疗/状态/胜负反馈。
4. 优化狗狗选择和模式大厅：减少机械网格感，增强桌面纸片与收藏卡气质。
5. 优化奖励和遗物：翻卡、盖章、贴纸和获得反馈。

## 验收标准

- 整体仍能一眼看出是当前 DogFight 的奶油手绘风，而不是新风格。
- 页面不再显得方正死板，主要入口和卡片具备自然的手绘纸片感。
- 商店、背包和战斗中的关键操作都有明确、可感知的反馈。
- 战斗过程能清楚看出谁投骰、哪个装备触发、打到了谁、造成了什么状态变化。
- 手绘细节不会影响文字可读性、点击区域、拖拽判断和移动端布局。
- `npm run test` 和涉及可玩版本的 `npm run build` 在实现后通过。
