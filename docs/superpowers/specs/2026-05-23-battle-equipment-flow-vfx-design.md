# 战斗装备流光特效强化设计

## 目标

本轮只强化战斗回放中的装备触发特效。玩家应能清楚看到“哪件装备触发、效果从哪里飞出、飞向谁或哪件装备、命中后产生什么结果”，同时整体表现要比当前更炫酷、更有冲击力。

不改战斗规则、装备数值、遗物数值、后端结算逻辑和回放播放节奏。本轮不处理商店购买、出售、升级、奖励领取等非战斗操作反馈。

## 推荐方案

采用方案 B：前端真实锚点定位流光，必要时补充少量展示用事件字段。

现有战斗回放已经有 `createBattlePresentation`、`BattleFxStage` canvas 特效层、装备触发高亮和目标狗狗高亮。新方案在这些基础上扩展，不重写整套战斗系统：

- 源点优先取当前触发装备卡片的真实 DOM 中心。
- 伤害、中毒、虚弱、冻结等负面效果飞向敌方头像或敌方状态区。
- 治疗、护盾、荆棘、激昂、净化等正面效果飞向自己的头像、血条或状态区。
- 影响装备时飞向目标装备卡；能精确推导目标装备的效果使用装备卡锚点，不能稳定推导时降级为飞向对应装备栏并高亮相关区域。
- `prefers-reduced-motion` 下保留颜色、目标高亮和命中闪光，取消长距离飞行轨迹。

## 锚点系统

在战斗 UI 内建立一组稳定锚点，不改变布局尺寸和点击命中区域。

### 装备锚点

`BattleEquipmentRow` 为每件战斗装备增加定位标记：

- `data-battle-anchor="item"`
- `data-battle-side="player | opponent"`
- `data-battle-item-id="<item.id>"`

当前触发事件有 `itemId` 时，流光源点取对应装备卡中心。装备触发高亮继续使用现有 `battle-item-trigger`，并叠加更强的边缘闪光和短促脉冲。

### 角色与状态锚点

`BattleDog` 增加以下定位标记：

- 头像：`data-battle-anchor="dog-avatar"`
- 血条：`data-battle-anchor="hp"`
- 正面状态区：`data-battle-anchor="status-positive"`
- 负面状态区：`data-battle-anchor="status-negative"`

不同效果按表现语义选择目标：

- `damage`、`miss`：目标头像。
- `heal`：己方头像和血条之间。
- `shield`：己方血条。
- `poison`、`weak`、`freeze`：敌方负面状态区，缺少状态区元素时用头像。
- `thorns`、`utility` 中的正面增益：己方正面状态区，缺少状态区元素时用头像。

## 目标装备识别

现有 `BattleEvent` 没有通用的 `targetItemId`，所以目标装备分两类处理。

### 可精确定位

以下效果可以在展示层或后端事件中稳定得到目标装备：

- `POISON_AND_DISABLE_RIGHTMOST`：目标为敌方最右侧装备。
- `TRIGGER_ADJACENT`、`ADJACENT_ON_EXTRA_ROLL`：目标为触发装备相邻装备。
- 附魔 `TRIGGER_NEIGHBOR`：目标为左侧、右侧或相邻装备。
- 附魔 `BUFF_NEIGHBOR_EFFECT`：目标为符合基础效果类型的相邻装备。
- 附魔 `GRANT_NEIGHBOR_EFFECT`：目标为被授予效果的相邻装备。
- `GRANT_LIFESTEAL_ADJACENT` 开场光环：目标为被授予吸血的相邻装备。

实现时优先在前端从快照和事件文本对应的装备规则推导目标。若推导不够稳定，再在 `BattleEvent` 增加展示字段：

- `targetItemIds?: string[]`
- `targetItemOwner?: "player" | "opponent"`

这些字段只服务回放表现，不参与战斗结算。

### 降级定位

无法可靠定位单件装备时，不硬指错误目标。表现降级为：

- 流光飞向目标方装备栏中心。
- 当前触发装备继续强高亮。
- 目标装备栏出现短促区域扫描光。
- 日志和目标角色仍按当前事件类型高亮。

## 视觉表现

特效分为三层，全部绘制在现有 `BattleFxStage` canvas 与少量 CSS 高亮中。整体观感要从普通连线升级为“流星式流光”：有明亮核心、有长拖尾、有星屑、有命中闪爆，玩家一眼能感到装备效果真的飞了出去。

### 主流光

从源锚点到目标锚点绘制贝塞尔曲线。曲线表现为流星弹道：

- 前端是高亮光核，中心接近白色，外层按效果类型染色。
- 光核后方保留 2-3 层不同长度的透明拖尾，形成明显速度感。
- 拖尾要有粗细变化，靠近光核更亮更粗，尾端逐渐变细变淡。
- 飞行中可加入轻微弧线和亮度脉冲，避免像静态直线。
- 飞行时间约 420-700ms，随战斗倍速缩短，但 2x/4x 下仍要保留可辨识的亮点和尾迹。

### 粒子尾迹

沿主路径产生星屑和余辉粒子，粒子数量比当前更丰富，但生命周期短，不能遮住装备名称、血量和状态。粒子形态按效果变化：

- 伤害：红金斩击碎片和短线。
- 治疗：绿色回旋光点和柔和圆环。
- 护盾：蓝色盾片、弧形碎光。
- 中毒：绿色雾点和气泡。
- 虚弱：紫色波纹、下坠碎纸感。
- 冻结：冰蓝碎光和细霜线。
- 装备影响：金色或蓝紫色流光锁链，落点是装备卡。

流星尾迹的通用要求：

- 光核附近生成高亮星点，颜色可接近白色或浅金色。
- 拖尾中段生成少量同色粒子，沿路径反向飘散。
- 尾端粒子透明度快速衰减，避免画面长期残留。
- 同一事件内粒子随机性要稳定，不因为重绘造成闪烁杂乱。

### 命中爆点

到达目标后触发短促闪爆，类似流星撞击目标后的光晕扩散：

- 头像目标：头像闪光、轻微受击或增益脉冲。
- 血条目标：血条边缘光圈和数字弹出。
- 状态目标：对应状态芯片闪亮。
- 装备目标：目标装备卡边缘扫光、轻微弹起、短暂连线残影。
- 高品质装备触发时，源装备和命中点可以额外出现一圈金色或钻石色星芒，但不能改变战斗结果含义。

## 数据流

1. 战斗回放推进到当前 `BattleEvent`。
2. `createBattlePresentation(event)` 生成表现类型、源语义和目标语义。
3. 新增的锚点解析函数读取战斗舞台 DOM，解析源元素和目标元素。
4. `BattleFxStage` 根据元素中心坐标绘制飞行特效。
5. CSS class 同步驱动装备、头像、血条、状态芯片或目标装备的命中反馈。
6. reduced motion 用户跳过长距离轨迹，仅保留源点闪烁、目标闪烁和结果数字。

## 组件边界

- `src/feedback.ts`：继续负责事件到表现语义的映射，可补充目标锚点类型，但不依赖 DOM。
- `src/App.tsx`：负责战斗 UI 锚点标记、目标装备推导、canvas 坐标解析和绘制。
- `src/App.css`：负责源装备、目标装备、目标角色、状态芯片的高亮动画。
- `src/server/game/types.ts` 与 `src/server/game/battle.ts`：只有在前端无法稳定推导目标装备时，才增加纯展示字段。

## 测试策略

先写测试再改生产代码。

- `src/feedback.test.ts`：覆盖伤害、治疗、护盾、负面状态、装备影响事件映射到正确目标语义。
- `src/App.structure.test.ts`：覆盖战斗装备、头像、血条、状态区存在稳定 `data-battle-anchor` 标记。
- `src/App.css.test.ts`：覆盖新增流光、源装备、目标装备、目标头像、状态芯片和 reduced motion CSS hook。
- 若新增 `targetItemIds` 字段，则在 `src/server/game/*.test.ts` 中覆盖至少一个精确目标装备事件。

## 验收标准

- 玩家能看出当前是哪件装备触发。
- 伤害或负面效果能明显飞向敌方头像或状态区。
- 治疗、护盾或正面增益能明显飞向自己头像、血条或状态区。
- 影响装备时，能精确定位的事件必须飞向目标装备卡。
- 不能精确定位的事件不能误导玩家，必须降级为装备栏区域反馈。
- 特效必须明显更闪耀，主轨迹要有流星般的高亮光核、长拖尾和星屑。
- 命中目标时必须有可见闪爆或光晕扩散，不能只是普通线条消失。
- 特效更炫酷，但不遮挡核心血量、装备名称和继续按钮。
- 移动端和桌面端布局不发生跳动。
- 本轮实现完成后必须运行 `npm run build`，确认单文件可玩版本重新生成。

## 非目标

- 不改装备、遗物、职业道具和战斗数值。
- 不更新外部数值 Excel，因为本轮不改变平衡模型。
- 不新增新的战斗事件类型。
- 不重做战斗 UI 布局。
- 不处理商店、背包、升级、出售等非战斗反馈。
