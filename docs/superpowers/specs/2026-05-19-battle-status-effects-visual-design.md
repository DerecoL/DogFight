# 战斗状态与特效可视化设计

## 背景

当前战斗回放已经能播放道具、投掷、伤害、治疗和中毒结算事件，但状态信息不够明确。玩家很难在战斗过程中判断：

- 哪些正面效果正在自己或对手身上生效。
- 哪些负面效果正在等待结算。
- 中毒、虚弱、冻结、失效等状态的层数、剩余时间或触发时机。
- 某次伤害是普通攻击、毒伤、反伤、护盾吸收还是控制效果带来的结果。

本设计目标是让战斗状态成为一套统一的可视化系统，而不是只给单个毒效果加特殊显示。

## 设计目标

1. 玩家能一眼区分正面效果和负面效果。
2. 中毒伤害必须明显不同于普通伤害：使用绿色数字、毒雾粒子和预损血条。
3. 多状态同时存在时，角色卡高度保持稳定，不挤压战斗主体。
4. 每个状态都能显示必要信息：层数、数值、剩余时间或剩余次数。
5. 即时效果和持续状态都要有对应表现，包括颜色、短动画、日志高亮和粒子效果。

## 总体布局

采用“角色内嵌状态条”方案，并将正负状态按血条分层：

- 血条上方显示正面效果。
- 血条本体显示生命、护盾覆盖和预损血量。
- 血条下方显示负面效果。

每个角色固定三层结构：

```text
角色名 / 当前生命
正面状态行：护盾、荆棘、额外投掷等
血条：当前生命 + 护盾覆盖 + 预损提示
负面状态行：中毒、冻结、虚弱、失效等
```

## 多状态显示规则

每条状态行最多常驻显示 3 个状态。超出后显示 `+N` 折叠入口。

状态排序规则：

- 正面效果：护盾/减伤类 > 荆棘反伤 > 攻速/额外投掷 > 其他增益。
- 负面效果：即将造成伤害的中毒 > 控制类冻结 > 失效 > 虚弱 > 其他异常。
- 刚新增、刷新或即将结算的状态临时顶到所在行最前，并播放短闪光。

折叠入口规则：

- 桌面端：悬浮或点击 `+N` 展开完整状态列表。
- 移动端：点击 `+N` 展开，点击外部关闭。
- 展开列表显示完整状态名称、层数/数值、剩余时间/次数。

## 状态字段

战斗事件需要携带双方当前状态快照。建议每个事件都包含：

```ts
type BattleStatusSnapshot = {
  poison?: { stacks: number; nextTickIn: number; tickDamage: number }
  weak?: { stacks: number }
  freeze?: { remaining: number }
  disabled?: { count: number }
  shield?: { amount: number }
  thorns?: { stacks: number }
  extraRoll?: { count: number }
}
```

事件层字段建议：

- `playerStatuses`
- `opponentStatuses`
- `previewHpDelta`：下一次已知持续伤害造成的预损血量。
- `statusChanged`：本事件新增、刷新、消耗或结算的状态列表，用于 UI 高亮。

## 视觉语言

状态颜色：

- 中毒：绿色，用于毒层、毒伤数字、毒雾粒子和预损血条。
- 护盾：蓝色，用于护盾值、护盾覆盖层和护盾吸收特效。
- 荆棘：金色，用于反刺闪光和反伤日志。
- 冻结：冰蓝色，用于霜裂、短暂停顿和控制状态。
- 虚弱：紫色，用于下坠波纹和伤害削弱提示。
- 失效：灰色，用于断电闪烁和被跳过的装备提示。
- 治疗：绿色偏亮，保持和毒伤不同的正向语义，使用 `+` 数字。

## 毒伤表现

中毒是首批重点状态，表现规则如下：

1. 负面状态行显示 `毒 X层 · Ys`。
2. 血条上显示预损段，表示下一次毒跳会扣掉的生命。
3. 毒伤结算时显示绿色 `-X` 数字。
4. 毒伤粒子使用绿色毒雾扩散，不使用普通红色受击粒子。
5. 战斗日志中毒伤事件用绿色高亮。
6. 叠毒或刷新时，毒状态芯片短暂闪光并移动到负面状态行最前。

## 其他状态表现

正面效果：

- 护盾：血条外层或底层显示蓝色覆盖；数值变化时显示 `盾+X` 或 `盾-X`。
- 荆棘：正面状态行显示层数；反伤时播放金色反刺粒子。
- 额外投掷/攻速：正面状态行显示剩余次数或短时增益；触发时骰子区播放快速闪动。

负面效果：

- 虚弱：负面状态行显示层数；下一次伤害被削弱时播放紫色下坠波纹。
- 冻结：负面状态行显示剩余时间；冻结期间角色图像叠加冰蓝裂纹或低饱和暂停效果。
- 失效：负面状态行显示剩余次数；装备被抵消时对应装备灰色闪烁。

## 前端组件调整

建议新增或拆分以下 UI 单元：

- `StatusEffectRow`：负责正面/负面状态行、排序、折叠和高亮。
- `StatusChip`：负责单个状态的颜色、文本、倒计时和变化动画。
- `HealthBar`：负责生命、护盾覆盖和预损血量。
- `BattleEffectBurst`：负责伤害、治疗、护盾、毒伤等数字跳字。

现有 `BattleDog` 可以继续作为承载组件，避免重做战斗页结构。

## 后端与单文件版同步

服务端 `simulateBattle` 和单文件可玩版内嵌 mock 战斗逻辑都需要输出同样的状态快照字段。

因为单文件版依赖 `npm run build` 重新生成：

- 任意影响状态、战斗事件、前端展示或打包逻辑的实现变更后，必须运行 `npm run build`。
- 构建成功后交付的单文件为 `E:\AI-GPT\DogFight\dist-click\DogFight-standalone.cmd`。

## 测试计划

需要覆盖以下行为：

1. 中毒施加后，战斗事件包含毒层、下一跳时间和下一跳伤害。
2. 毒伤结算事件使用 `effectType: 'POISON'`，并能生成绿色毒伤表现。
3. 多个正面状态只出现在血条上方。
4. 多个负面状态只出现在血条下方。
5. 超过 3 个状态时显示 `+N`，完整列表可展开。
6. 单文件可玩版也能显示职业毒、普通毒、护盾、虚弱等状态。
7. `npm run build` 成功并重新生成 `dist-click\DogFight-standalone.cmd`。

## 非目标

- 本次不重新设计整个战斗页面布局。
- 本次不增加新的战斗规则或新状态类型。
- 本次不改变现有装备、遗物、职业奖励的数值。
- 本次不把状态系统做成独立图鉴页面。
