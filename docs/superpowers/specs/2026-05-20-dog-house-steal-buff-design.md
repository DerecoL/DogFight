# 小狗窝偷取增益设计

## 目标

为通用装备“小狗窝”新增触发效果：触发时保留原本治疗，并从敌方偷取 1 层真正的正面增益。同步修正规则定义：护盾是特殊血量，不再属于正面增益。

## 规则定义

- 小狗窝仍在掷出 1/2 点时触发，基础效果仍为恢复 12 点生命。
- 小狗窝触发后，若敌方存在可偷取增益，则偷取 1 层给自己。
- 可偷取增益只包括层数型增益：优先【荆棘】，其次【加速】。
- 敌方只有护盾时，小狗窝不偷取任何效果。
- 护盾继续作为特殊血量吸收伤害、显示护盾值和护盾条，但不出现在“正面增益”列表中，也不被偷取、清除或净化增益类效果影响。

## 战斗事件

- 成功偷取时追加一条装备事件，例如：`小狗窝 偷取 1 层【荆棘】`。
- 没有可偷取增益时不追加额外事件，避免战斗日志噪音。
- 偷取事件不改变双方生命值，只改变双方对应状态层数。

## 数据与展示

- `dog-house` 增加高级效果标识 `STEAL_ENEMY_BUFF`。
- 小狗窝描述改为“恢复生命，并偷取敌方 1 层增益（荆棘优先，其次加速；护盾不算增益）”。
- 战斗 UI 仍通过独立的 `playerShield` / `opponentShield` 字段显示护盾，不依赖正面状态列表显示护盾。

## 测试要求

- 小狗窝能从敌方偷取 1 层【荆棘】，敌方减少 1 层，己方增加 1 层。
- 小狗窝能在敌方没有荆棘但有加速时偷取 1 层【加速】。
- 敌方只有护盾时，小狗窝不偷取，护盾值保持不变。
- 护盾不再出现在 `positive` 状态快照中，但仍通过 `playerShield` / `opponentShield` 记录和吸收伤害。
- `npm run build` 必须成功，以重新生成单文件版本。
- 外部数值表需要同步小狗窝描述和“护盾不是增益”的规则口径。
