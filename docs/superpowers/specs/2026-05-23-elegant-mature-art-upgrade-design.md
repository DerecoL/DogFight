# DogFight 手绘美术精修提升方案

## 目标摘要

本次方向必须继续坚持手绘风。目标不是把 DogFight 改成写实、暗黑、硬核或“高级但冷”的游戏，而是在现有奶油纸面、手绘描边、桌游卡片感的基础上，把画面做得更精致、更丰富、更有材质、更有明暗层次和特效表现。

新的视觉关键词：

- 手绘但不粗糙
- 可爱但不廉价
- 纸面但有厚度
- 卡片但有收藏质感
- 战斗但有光影和特效
- 明亮但有明暗过渡

最终画面应像一本会动的手绘桌游绘本：每个按钮、格子、装备、狗狗、遗物、战斗反馈都保留手画痕迹，但细节更完整，材质更真实，动作更有戏剧性。

## 当前状态判断

项目已经具备手绘风基础：

- `src/App.css` 中已有奶油纸面、手绘描边、纸张纹理、贴纸卡片、手绘阴影、战斗舞台、状态 VFX 和按钮反馈。
- `src/App.tsx` 中已有视觉 class 钩子，如 `paper-card`、`sketch-panel`、`paper-shop-card`、`paper-inventory`、`handdrawn-stage`、`handdrawn-dice`、`handdrawn-result`。
- 资产目录已有狗狗图片、装备 SVG、遗物 SVG、背景图和音乐。

当前问题不是风格错，而是精细度不够：

- 纸面材质还偏平，缺少厚度、边缘磨损、笔触和局部阴影。
- 卡片和面板有手绘外观，但层次还不够，很多元素像贴在同一个平面上。
- 装备和遗物图标缺少统一的手绘底座、描边、投影和品质表现。
- 明暗过渡偏弱，页面缺少“光从哪里来”的一致感。
- 战斗特效有反馈，但还不够像手绘动画，缺少轨迹、爆点、残影、墨迹、纸屑和状态叠层。
- 不同页面已有装饰，但细节密度不均，有些地方精致，有些地方仍像普通网页 UI。

## 核心美术方向

### 1. 强化手绘材质，而不是降低手绘感

所有主要元素继续走手绘纸面方向，但要补上“材质真实感”：

- 纸张：增加纤维、颗粒、压痕、边缘暗线、折角、轻微污渍和局部磨损。
- 墨线：描边不应完全机械等宽，可通过多层 border、伪元素、SVG noise 或 box-shadow 模拟手绘线条粗细变化。
- 颜料：按钮、卡片、状态和品质色使用水彩晕染、彩铅填色、马克笔叠色，而不是纯色块。
- 阴影：所有可拿起的卡片、道具、奖励牌都要有接触阴影和悬浮阴影两层。
- 高光：装备、遗物、品质边框使用手绘高光短线，不使用强烈霓虹发光。

目标是“画出来的细节更多”，不是“装饰更多”。细节必须服务识别和质感。

### 2. 建立边框、角花和纹样系统

需要把“边角装饰”作为这次美术升级的核心，不再只靠普通圆角卡片和简单描边。参考用户给的示例，主要元素应具备厚黑手绘外框、内侧角花、卷草纹、短线磨损、粗细变化和纸面纹理。

边框要求：

- 外框应明显更厚，主卡片和战斗面板使用 3-5px 的黑棕色手绘粗描边。
- 描边不能完全等宽，四边要有轻微波动、断点、墨迹堆积和边缘抖动。
- 单根线条需要有粗细变化：线头可细，转角和局部压笔处更粗。
- 外框之外可增加一层浅色柔光或纸边高光，让卡片从背景中浮起来。
- 内框可用更细的手绘线，形成“粗外框 + 细内框 + 角花”的层次。

角花和花纹要求：

- 卡片四角至少有一种手绘角花：L 形折角、卷草纹、爪印纹、骨头纹、短横线、墨点簇。
- 重要卡片可使用更完整的角花组合：左上/右上为内角粗线，底部两侧为卷草或波浪纹。
- 战斗、遗物、奖励、狗狗卡、商品卡应使用不同纹样主题，但共享同一笔触语言。
- 花纹应贴边或压在边角，不要放在正文阅读区域。
- 小卡片可以只保留角线、短线和墨点；大面板可以加入横向装饰线、卷草纹和底角花。

纹理要求：

- 红、黄、蓝、绿等底色都不能是纯色块，应叠加低透明度纸纹、笔刷颗粒、斜向纤维或轻微水彩噪声。
- 可使用 CSS radial-gradient、linear-gradient、mask、伪元素或小型 SVG pattern 实现纹理。
- 纹理强度要可控，不能影响文字对比度。

实现目标：

- 任何主卡片截图都应能看出“这是画出来的边框”，而不是浏览器默认边框。
- 边角细节要成为 DogFight 的视觉签名之一。

### 3. 建立全局手绘 HUD 和标题横幅

除了单张卡片，整个界面也需要有手绘游戏框架感。参考示例中的顶部状态栏、横向粗黑分隔线、卷草角饰、图标槽和红色横幅标题，DogFight 应建立统一的 HUD 装饰语言。

全局 HUD 要求：

- 顶部栏不应只是普通白色信息条，应像一条手绘状态卷轴或纸板横梁。
- 顶部栏可以使用厚黑底边、局部缺口、卷草角饰、短线磨损和纸面颗粒。
- 资源、生命、金币、胜负、回合等信息可以放入小型手绘图标槽，图标槽使用厚描边、内阴影和纸面底色。
- 模式、回合、奖励、商店类型等标题应使用手绘横幅或缎带形标题牌，而不是普通居中文本。
- 横幅两端可有折叠阴影、缝线、短排线和手绘卷边。
- 页面大区之间可用粗黑手绘分隔线，而不是细灰线。

界面框架要求：

- 主页面边缘可有轻微暗角、外圈纸张边框或粗黑底线，增强“画面被画框包住”的感觉。
- 空白区域应有低透明度纸纹、刷痕、水彩云影或铅笔短线，避免大面积平底色。
- 图标槽、装备槽、遗物槽和奖励槽应共享“厚描边小方框”的手绘风格。
- 重要操作按钮可以像示例中的蓝色手绘按钮：粗描边、内部纹理、按下时纸片下沉。

实现目标：

- 进入任意主页面，都能看到全局统一的手绘框架，而不是只有卡片局部有风格。
- 标题、状态栏、资源条、按钮、图标槽和卡片边框应属于同一套笔触语言。

### 4. 建立统一明暗体系

当前画面整体偏亮，缺少空间层次。建议建立统一光照：

- 默认光源来自左上方。
- 卡片右下方有接触阴影。
- 面板边缘内侧有轻微暗边，表现纸张厚度。
- 重要可点击元素有柔和上亮下暗的手绘渐变。
- 战斗舞台中央更亮，边缘略暗，形成剧场聚光。
- 弹出的浮层和提示应比背景更亮，底部阴影更明显。

明暗处理重点：

- 不要把整体变暗。
- 不要大面积黑色遮罩。
- 使用暖棕、浅橙、纸灰、淡蓝灰做阴影。
- 用局部暗角、投影、边缘压痕和聚光制造层次。

### 5. 提高手绘细节密度

每类元素都需要有专属手绘细节：

- 按钮：纸片压下、边缘墨线、轻微颜料堆积。
- 卡片：纸纹、厚外框、细内框、角花、卷草纹、角落磨损、手绘标题条、品质角标、底部短阴影。
- 格子：木板/纸板凹槽、浅阴影、可放置时绿色手绘圈线。
- 装备：统一图标底座、手绘投影、品质边框、触发时描边闪动。
- 遗物：更像特殊收藏章，增加底座、铭牌、轻微神秘纹理。
- 狗狗：头像周围增加舞台投影、状态叠层、受击/治疗/护盾的局部手绘效果。
- 背景：不能只是纯色，应有纸纹、桌面纹、轻微笔刷和视觉暗角。

### 6. 加入手稿风格层

在手绘卡牌和纸面材质之上，增加一层“手稿/草图/设计稿”痕迹，让界面更像一本正在被翻阅和标注的手绘游戏手册。手稿风格不是潦草，而是有控制地保留创作痕迹。

手稿元素：

- 草图辅助线：低透明度铅笔线、构图线、短横线、测量线，主要放在背景、卡片边缘和面板空白处。
- 手写批注：少量小字标记、箭头、圈选、划线，用于强调关键状态，例如可升级、已拥有、触发来源。
- 涂改痕迹：轻微擦除印、重复描线、局部修正线，让卡片边缘和标题牌更像手工绘制。
- 设计稿符号：小圆点、定位十字、虚线框、角落编号、墨水试笔线，可用于装备槽、遗物槽和奖励牌边缘。
- 未完成线稿：背景中可以有淡淡的狗爪、骨头、骰子、卷草、装备轮廓草图，作为低透明度纹理。

使用原则：

- 手稿层必须低透明度，优先出现在边缘、背景和空白处。
- 不要覆盖正文、数值、按钮文字、血条数字和装备描述。
- 功能提示可以使用手稿符号，但不能只靠装饰表达状态，仍要保留清晰的颜色和布局反馈。
- 同一屏手稿元素要有节制，避免变成脏乱涂鸦。

### 7. 做到位的交互动效与选择反馈

所有可选择、可点击、可拖拽、可购买、可领取的元素都需要有明确反馈。动效方向是“纸片、卡牌、手绘物件在桌面上被按下、弹起、拿起、放下”，而不是普通网页按钮变色。

基础反馈要求：

- 悬停：卡片轻微抬起，阴影变深，边框墨线略微变清晰。
- 按下：按钮或卡片向下压 1-3px，阴影收短，松开时小弹跳回位。
- 选中：选中卡片应有 120-220ms 的小弹跳、边框重描、角花亮一下或手绘圈线收拢。
- 取消选中：不应瞬间消失，边框和高光应快速淡出，纸片轻微回落。
- 禁用/不可选：不只降低透明度，应有灰色水洗、铅笔斜线或轻微“按不动”的短反馈。

操作反馈要求：

- 购买成功：商品卡轻弹或向背包方向短移，价格牌跳动，金币数字有轻微扣减反馈。
- 金币不足：商品卡或价格牌小幅左右抖动，红色铅笔短线闪一下。
- 拖拽拾起：物品卡离桌，旋转 1-3 度，接触阴影变成悬浮阴影。
- 放置成功：纸片落桌，短回弹，阴影收缩，目标格边缘亮一下。
- 放置失败：物品回弹，目标格出现红色叉线和短抖动。
- 出售成功：物品缩入出售区，出现小票、金币小跳或手写“售出”纸条。
- 升级成功：两张/目标装备短聚拢，边框重新描一圈，品质角标弹出。
- 领取奖励：被选奖励卡抬起并收入目标区，未选卡轻微淡出或滑走。

动效曲线要求：

- 使用轻微 overshoot 和 ease-out，避免线性生硬移动。
- 小弹跳要短，通常 120-260ms，不能拖慢高频操作。
- 大型奖励/胜负演出可以更长，但应控制在 400-700ms。
- `prefers-reduced-motion: reduce` 下保留颜色、描边、阴影和静态选中态，禁用大位移和循环动画。

验收目标：

- 玩家每次点击、选择、购买、放置、领取都能感到“操作被接住了”。
- 反馈要有手绘纸片感，不是普通网页 hover/active。

### 8. 特效必须像手绘动画

特效方向应区别于传统粒子特效。不要做科幻激光、强霓虹爆炸，而是做手绘动画感：

- 伤害：红色手写数字、短促墨线爆点、纸屑或划痕。
- 治疗：绿色水彩圆环、柔和叶片线、向上漂浮的小笔触。
- 护盾：蓝色半透明手绘泡泡、铅笔弧线、碎裂时纸片裂纹。
- 中毒：草绿色墨雾、斑点扩散、血条预损区域有毒雾纹理。
- 虚弱：紫色下坠波纹、淡淡晕影、攻击时出现“被削弱”的手写小标。
- 冰冻：浅蓝霜边、短暂停顿、纸面冰晶线，不做硬科幻冰块。
- 荆棘：金棕色尖刺短线、反伤时从狗身上弹出小刺。
- 爆发：手绘星芒、冲击线、墨点扩散、短暂纸面震动。

每个特效应有三段：

1. 起点：装备或状态源头亮起。
2. 过程：手绘轨迹飞向目标。
3. 结果：目标身上出现受击、治疗、护盾或状态变化。

## 分模块提升方案

### 模式大厅

目标：保持轻松手绘入口，但让第一屏更完整、更像游戏封面页。

改法：

- 背景增加手绘桌面、纸纹暗角、散落铅笔线和低透明度狗爪印。
- 背景和卡片空白处加入低透明度手稿线：狗爪草图、骰子构图线、铅笔箭头和轻微擦除痕迹。
- 顶部品牌区改成手绘横幅标题，带粗黑描边、两端折叠、底部阴影和少量卷草纹。
- 模式卡做成不同形状的手绘纸牌，而不是完全一致的矩形卡；每张卡要有粗外框、细内框、角花和底角卷草纹。
- 每个模式入口增加小型手绘插图：休闲模式可用骨头/骰子，天梯可用奖杯，斗狗模式可用竞技旗，巅峰可用皇冠或山峰。
- 锁定/未开放状态使用手绘胶带、灰色铅笔线和淡化插图。
- 历史记录面板增加手写账本感：横线纸、勾选符号、奖章贴纸。
- 模式卡选择时必须有小弹跳和边框重描反馈，进入模式时卡片像被翻开或推入桌面。

验收标准：

- 仍然轻松可爱，但不空、不像普通卡片网格。
- 模式差异能通过图形和材质直接看出来。

### 选狗页

目标：让狗狗选择更像手绘角色册，增强角色魅力。

改法：

- 狗卡片增加手绘相框、局部阴影、纸面折角和专属小图案。
- 狗卡片边框使用更厚的手绘线条，四角加入不同犬种主题角花，例如爪印、骨头、毛发短线、皇冠卷纹。
- 狗狗详情页加入图鉴手稿感：淡铅笔轮廓、箭头批注、特性圈选、幸运数字旁的小手写标记。
- 头像区域增加浅色背景涂抹，不要让狗图直接浮在纯色块上。
- 选中时卡片轻微抬起，边缘出现手绘光圈、星芒和短线强调。
- 选中狗狗时必须有清晰小弹跳，狗狗头像轻微上浮，详情页像纸张翻页一样更新。
- 详情面板像翻开的图鉴页：左侧狗狗大图，右侧特性说明和幸运数字。
- 每种狗保留独特主色，但主色应像手绘颜料涂层，不是硬色块。

验收标准：

- 狗狗角色更有“被画进图鉴”的感觉。
- 卡片更精致，但文字仍清楚。

### 商店

目标：商店像手绘货架，不是商品按钮列表。

改法：

- 商店标题使用红色或暖色手绘横幅，类似“选择遗物”示例中的缎带结构，而不是普通 `h2` 文本。
- `shop-shelf` 增加木板/纸板货架底纹、手绘横线、局部暗角。
- 商品卡加入多层纸片结构：底纸、图标底座、标题条、价格吊牌、粗外框、细内框和角花装饰。
- 商品卡底部或上边缘增加类似示例的横向手绘装饰线：短线、波浪线、卷草纹、墨点簇。
- 商品卡空白边缘可加入轻微手稿标记，例如小箭头、试笔短线、已拥有数量旁的圈选痕迹。
- 价格标签改成手写挂牌，金币图标有小高光。
- 买得起和买不起不仅靠透明度：买不起时增加铅笔斜线、灰色水洗和价格牌红色短线。
- 刷新商店时商品有短暂翻牌/抽换纸片效果。
- 购买成功时商品卡向背包方向轻弹，或出现“已收入”的小纸条反馈。
- 选中商品时商品卡要有短弹跳、价格牌轻晃、边框墨线重新描一圈。

验收标准：

- 商品看起来是摆在手绘货架上的物件。
- 价格、品质、触发点数、效果说明更清晰，不被装饰淹没。

### 背包与装备栏

目标：高频拖拽区更有材质，但命中区域稳定。

改法：

- 格子从普通边框升级为手绘托盘凹槽：内阴影、纸板纹、边缘暗线。
- 装备栏和背包大面板四角增加明显角饰，外圈使用更粗的黑棕描边，内圈用短断线模拟手画边。
- 格子边缘可加入短刻痕、细小墨点和压痕线，但格子内部保持干净，避免影响道具识别。
- 可升级、可出售、可放置等状态可以叠加手稿符号：铅笔圈、箭头、虚线框、短批注。
- 可放置区域使用绿色手绘圈线和柔和水彩底色。
- 非法区域使用红色铅笔叉线和轻微抖动。
- 拖拽物品增加离桌效果：更深投影、轻微旋转、底部半透明占位影。
- 放下物品时有短促“纸片落桌”动效：回弹、阴影收缩、边缘亮一下。
- 点击选中装备时要有小弹跳和手绘圈线反馈，避免只靠选中边框。
- 可升级物品使用小型手绘角标，不要只靠星星堆叠。

验收标准：

- 操作区看起来更像实体桌面。
- 拖拽、放置、出售、升级的状态一眼可辨。
- 布局尺寸不因为装饰变化。

### 战斗页

目标：战斗页是最大美术提升点，要从“战斗信息面板”升级成“手绘小剧场”。

改法：

- 战斗顶部 HUD 使用手绘状态栏，资源、血量、回合、速度按钮都放入厚描边小槽位，形成类似示例顶部工具条的游戏框架。
- 舞台背景增加手绘场景层：纸板地台、浅色聚光、前后景阴影。
- 战斗舞台外框应最有“手绘卡牌”特征：厚黑不规则描边、四角粗笔压痕、内侧装饰线、底角卷草或爪印花纹。
- 战斗背景可加入淡淡的手稿战术线：虚线箭头、目标圈、轨迹预演线和被擦掉的旧线条。
- 两只狗脚下增加接触阴影和椭圆舞台光。
- 血条从网页进度条改成手绘生命牌：纸质外框、内部水彩填充、受伤时红色边缘抖动。
- 骰子区强化为战斗节奏中心：骰子滚动时有手绘残影，落点有纸面冲击线。
- 装备触发必须有因果链：装备描边亮起 -> 手绘轨迹 -> 目标反馈 -> 日志高亮。
- 战斗速度按钮、日志展开、继续按钮都要有纸片按压和小弹跳反馈，不能像静态表单按钮。
- 浮字使用手写风格，大小和颜色按效果区分，出现后有短暂墨迹扩散。
- 胜负结果做成手绘战报牌，翻出时有纸屑、盖章或手写判定。

特效细节：

- 攻击轨迹用红棕色速度线和墨点。
- 治疗轨迹用绿色柔光水彩线。
- 护盾轨迹用蓝色弧线和半透明泡泡。
- 毒用绿色雾点叠在目标和血条预损区域上。
- 虚弱用紫色下坠线和淡淡暗影。
- 冰冻用浅蓝霜边和短暂停顿感。
- 反伤用从目标身上弹回的金棕色短刺线。

验收标准：

- 不看日志也能理解大多数战斗因果。
- 战斗特效明显更丰富，但仍像手绘，不像科幻粒子。
- 桌面端和移动端都不能遮挡血条、骰子、装备和继续按钮。

### 遗物与奖励

目标：遗物和奖励要更有手绘收藏感。

改法：

- 奖励页标题使用手绘横幅，横幅两端有折叠阴影和粗黑描边，位置居中但不使用普通网页标题样式。
- 遗物槽像手绘展示架：每个槽有纸板凹槽、暗边和空槽标记。
- 遗物展示架外框要有特殊花纹，区别于普通装备：可使用卷草纹、神秘螺旋、短符号线和四角徽章纹。
- 遗物和奖励牌可加入“研究手稿”痕迹：小编号、虚线框、局部圈选、神秘符号草图。
- 遗物图标增加统一底座、手绘高光和品质边。
- 遗物升级时边框像被重新描了一圈，出现短暂墨线流动。
- 三选一奖励像从纸盒里抽出三张卡，选中后有收入遗物栏/背包的动效。
- 奖励卡选择必须有明确反馈：选中卡弹起、边框重描、角花亮起，确认领取时收入目标区。
- 职业奖励仪式页增加舞台光、纸屑、手绘徽章和狗狗主题装饰。

验收标准：

- 遗物看起来比普通装备更稀有、更像收藏品。
- 奖励选择有惊喜感，但不拖慢节奏。

## 资产与样式补充

建议新增或整理这些手绘资产：

- 纸张纹理：纤维、压痕、折角、边缘磨损。
- 桌面纹理：浅木纹、布纹、手绘暗角。
- 手绘边框：卡片边框、品质边框、奖励边框、血条边框。
- 角花纹样：L 形内角线、卷草纹、螺旋纹、爪印纹、骨头纹、短排线、墨点簇、底角大花纹。
- 横幅和 HUD：标题缎带、顶部状态栏粗黑底边、方形图标槽、资源槽、按钮槽。
- 手稿元素：铅笔辅助线、虚线框、圈选线、箭头批注、测量线、擦除痕迹、试笔短线、淡轮廓草图。
- 特效素材：速度线、墨点、纸屑、星芒、水彩圈、毒雾、霜边、护盾泡泡。
- 状态叠层：中毒、虚弱、护盾、冰冻、荆棘、狂怒。
- UI 装饰：价格吊牌、手写标签、胶带、折角、铅笔叉线、盖章。
- 动效 token：按压、回弹、轻弹跳、翻牌、落桌、拾起、收入、错误抖动、边框重描、角花闪动。

实现优先级：

- 能用 CSS 伪元素、渐变、滤镜、box-shadow 做的，优先用 CSS。
- 装备/遗物/状态图标需要统一视觉时，再补 SVG。
- 战斗轨迹和短生命周期特效可以继续使用 canvas，但绘制风格必须是手绘线、墨点和水彩圈。
- 避免新增大体积图片和复杂 3D。

## 验收标准

视觉验收：

- 画面仍然明确是手绘风，而且手绘感比现在更强。
- 每个主界面都有纸张、笔触、阴影、明暗和材质细节。
- 主要卡片和面板必须有可见的厚描边、粗细变化、角花或边缘纹样。
- 顶部 HUD、标题横幅、图标槽和按钮必须共享同一套手绘描边语言。
- 主要页面应出现适量手稿风格元素，例如铅笔线、批注、虚线框、圈选或擦除痕迹，但不能干扰正文阅读。
- 页面不再像平铺网页 UI，而像一套可操作的手绘桌游。
- 装备、遗物、狗狗、状态和奖励都有更明显的美术身份。
- 不出现写实硬核、暗黑压抑、赛博霓虹或纯扁平现代 UI 的偏移。

交互验收：

- 商店购买、拖拽放置、出售、升级、奖励选择、战斗播放都有更强反馈。
- 选择操作必须有可见小弹跳、边框重描或手绘圈线反馈，不能只改变颜色。
- 错误操作必须有短抖动、铅笔叉线或价格牌反馈，不能只弹 toast。
- 战斗中可以清楚看出“谁触发、轨迹到哪里、造成什么结果”。
- 高频操作区域不因为细节增加而降低可读性和可点击性。
- 移动端文本不溢出，不遮挡按钮、血条、格子和继续操作。
- `prefers-reduced-motion` 下仍保留静态层次，不依赖动画才能读懂状态。

技术验收：

- 修改前端展示后必须运行 `npm run build`，并确认 `E:\AI-GPT\DogFight\dist-click\DogFight-standalone.cmd` 重新生成。
- 如果只改文档，不需要运行 `npm run build`。
- 不涉及装备、遗物、战斗数值时，不需要更新 `C:\Users\User\Desktop\狗骰乱斗\狗骰乱斗.xlsx`。
- 建议更新 `src/App.css.test.ts` 和 `src/App.structure.test.ts`，锁定新的手绘材质、特效、明暗和布局要求。

## 明确不做

- 不削弱手绘风。
- 不转向写实、暗黑、赛博、像素、二次元卡牌或硬核军事风。
- 不把所有颜色压暗成“成熟风”。
- 不把手稿风做成脏乱涂鸦，也不让批注和草图线压住正文、数值或按钮。
- 不大规模重写 React 页面结构。
- 不修改战斗规则、装备数值、遗物数值或经济模型。
- 不为了装饰牺牲拖拽命中、移动端布局和文本可读性。

## 一句话方向

在 DogFight 现有手绘桌游风格上继续加厚：更多笔触、更多纸张和颜料材质、更清楚的明暗过渡、更有因果的手绘战斗特效，让它从“手绘网页游戏”提升为“精致、有细节、会动的手绘桌游”。
