# 背景音乐设计

## 目标

游戏接入用户提供的背景音乐 `The_Final_Inventory.mp3`。玩家进入已登录的游戏界面后，客户端默认尝试播放音乐；顶部栏提供一个明确的音乐按钮，允许玩家随时开启或关闭音乐。

## 范围

- 将 MP3 作为前端静态资源随游戏发布。
- 音乐在登录后的游戏壳层中循环播放。
- 默认偏好为开启音乐。
- 顶部栏显示音乐开关按钮。
- 玩家手动关闭音乐后保存偏好，刷新页面仍保持关闭。
- 不为登录页播放音乐。
- 不增加战斗音效、音量滑块、曲目列表或淡入淡出效果。

## 前端行为

新增背景音乐资源路径，例如 `/assets/audio/the-final-inventory.mp3`。`App` 在已登录界面创建一个可复用的音频控制逻辑，音乐元素设置为循环播放，并根据玩家偏好决定是否尝试播放。

默认情况下，玩家登录后进入 `Shell` 时音乐偏好为开启，客户端会调用 `audio.play()` 尝试播放。现代浏览器可能会因为页面还没有用户手势而拒绝自动播放；这种失败不显示错误，也不影响游戏流程。顶部栏按钮仍显示音乐为开启状态，玩家点击按钮后会再次触发播放。

当玩家点击音乐按钮关闭音乐时，客户端暂停音频并将偏好写入 `localStorage`。当玩家再次点击开启时，客户端恢复播放并更新偏好。退出登录或离开已登录界面时，音频暂停，避免登录页继续播放。

## 界面

顶部栏复用现有 `IconButton` 样式，放在退出登录按钮旁边。开启状态使用音乐图标，按钮标题表达“关闭音乐”；关闭状态使用静音图标，按钮标题表达“开启音乐”。按钮不占用额外面板，不改变主要游戏布局。

## 错误处理

自动播放失败只记录在内部状态中，不弹错误提示。手动点击播放仍失败时也不阻塞游戏，因为失败通常来自浏览器策略、设备音频权限或资源加载问题。音频资源加载失败时，按钮仍可点击，但游戏界面不显示全局错误。

## 测试

新增或扩展结构测试，覆盖以下内容：

- 存在背景音乐资源路径。
- `Shell` 接收音乐开关状态和切换回调。
- `TopBar` 渲染音乐按钮。
- 使用 `localStorage` 保存音乐偏好。
- 自动播放失败被捕获，不进入全局错误流程。

构建验证使用现有 `npm test` 和 `npm run build`。
