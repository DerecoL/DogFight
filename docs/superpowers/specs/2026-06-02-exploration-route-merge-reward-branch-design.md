# 探索地图路线汇合与少战收益旁路设计

## 背景

当前探索地图由 `src/server/game/map.ts` 生成 10 层节点。玩家战由 `playerBattleLayerSet()` 选出若干整层后统一替换为 `PLAYER_BATTLE`，因此任意完整路线遇到的玩家战数量相同。`connectMapLayers()` 只按相邻层的横向距离连接，没有专门把多条路线收束到玩家战节点。

这会带来两个问题：

- 路线缺少明显汇合点，玩家选择不同路线后很少感到路线重新交叉。
- 虽然地图整体有 3、4、5 次玩家战的随机密度，但同一张地图内每条路线的玩家战次数固定，缺少“某条路少打一场、换取额外地图收益”的惊喜。

## 目标

- 在玩家战节点上制造一部分 2 条或 3 条路线汇合。
- 小概率生成一条少一次玩家战的收益旁路。
- 收益旁路使用现有节点类型，不新增装备、遗物、图标或新 UI。
- 保持地图前进连通，所有完整路线都能从第 1 层走到第 10 层。
- 保持玩家战总节奏接近当前版本，避免收益旁路过度提高局内经济。

## 非目标

- 不新增地图节点种类。
- 不新增奖励结算接口。
- 不修改玩家战、野怪战、商店、事件的具体结算规则。
- 不调整装备、遗物、战斗参数或数值 Excel。
- 不重做地图 UI 布局。

## 推荐方案

采用“玩家战汇合点 + 小概率收益旁路”的生成策略。

地图仍按 10 层生成。玩家战层仍由现有随机逻辑决定，但生成节点时不再要求玩家战层的每个节点都一定是玩家战。系统会小概率选中一个玩家战层，把该层某一个非核心列替换成收益节点，形成少一次玩家战的旁路。旁路节点优先从 `EVENT`、`SHOP_EQUIPMENT`、`REST` 中选择：

- 首张地图不使用 `REST`，继续沿用当前首张地图无休息点的节奏。
- 后续地图可以使用 `REST`，但仍保持低概率。
- 旁路每张地图最多 1 个节点。

连接阶段会优先在玩家战层制造汇合：当下一层存在玩家战节点时，允许 2 条或 3 条上一层路线连到同一个玩家战节点。收益旁路会在下一层或后续玩家战点重新汇入主路线，确保它是“少打一场后拿收益”，而不是永久脱离主路线。

## 数据与兼容性

`ExplorationMapNode` 不需要新增字段。旁路直接表现为现有 `kind`。`nextNodeIds` 仍是唯一的路线连接数据。旧存档中的 `mapState` 可以继续通过 `normalizeExplorationMapState()` 使用，不需要迁移。

## 测试

实现前先补失败测试：

- 多个固定种子中至少存在玩家战节点有 2 或 3 个入边，证明路线能在玩家战汇合。
- 多个固定种子中至少存在一张地图，其完整路径的玩家战数量不完全相同，证明少战收益旁路存在。
- 所有生成地图仍满足相邻层前进、终点可达、每条完整路径玩家战数量不会低于安全下限。
- 首张地图仍不生成休息点。

实现后运行：

- `npx vitest run src/server/game/map.test.ts`
- `npm run build`
