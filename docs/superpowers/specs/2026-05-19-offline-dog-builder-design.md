# 离线狗构筑模拟器设计

## 背景

当前离线狗由 `src/server/state.ts` 中的 `seedGhost()` 生成。它只按回合把固定的普通装备塞进装备栏，不会像真实玩家一样选择商店、购买装备、升级品质、选择职业装备、选择遗物，也不会根据狗狗特性形成构筑方向。结果是离线对手强度偏低，且不同狗之间缺少可辨识的策略差异。

本设计采用“离线构筑模拟器”方案：服务器在没有真实 ghost 可匹配时，根据 `round`、`wins`、`losses` 和确定性随机种子，模拟一条简化但接近真实玩家的构筑路径，最终输出现有 `FighterSnapshot`。

## 目标

- 离线狗应更像真实玩家逐回合构筑出来的结果，而不是固定模板。
- 不同狗狗应体现自己的构筑偏好和职业装备联动。
- 离线狗会随回合自然变强，包括更换装备、升级品质、选择职业装备和遗物。
- 构筑结果必须确定性可复现：相同输入得到相同离线狗。
- 复用现有物品、职业装备、遗物、品质和格子规则，避免复制另一套战斗规则。

## 非目标

- 不实现完整 UI 商店流程，也不把离线构筑写入数据库。
- 不追求离线狗与人类玩家经济行为完全一致。
- 不改变 `simulateBattle()` 的输入输出结构。
- 不改变真实玩家的购买、移动、升级和匹配 API。

## 架构

新增 `src/server/game/offline-builder.ts`，提供纯函数：

```ts
export function buildOfflineFighter(input: OfflineBuildInput): FighterSnapshot
```

`seedGhost(round, wins, losses)` 保持现有公开行为，但内部改为调用 `buildOfflineFighter()`。这样 `/api/runs/:runId/battle/match` 的 fallback 逻辑不需要改接口。

离线构筑器包含三层：

1. **策略档案**：按狗狗类型定义偏好的装备标签、商店类型、职业装备、遗物和幸运数字策略。
2. **回合模拟**：从初始 starter 装备开始，逐回合获得预算、选择补强、升级、摆放，并在第 3/6 回合加入职业装备。
3. **快照输出**：把模拟后的装备、遗物和狗狗信息转换为 `FighterSnapshot`。

## 策略档案

每只狗有一个 `OfflineBuildProfile`：

```ts
type OfflineBuildProfile = {
  dogType: DogType
  shopPreference: ShopType[]
  itemTags: string[]
  classRewards: string[]
  relics: string[]
  keepStarterDice?: number[]
  luckyNumber?: (seedState) => number
}
```

初始策略如下：

- 柴犬：偏 `SMALL`、`SMALL_DICE`，优先小件、1-3 点、相邻触发和额外投掷职业装备。
- 萨摩耶：偏 `BIG_DICE`、`MEDIUM`，优先 4-6 点、恢复、弱化、半骰和点金遗物。
- 土狗：偏 `GENERAL`、`MEDIUM`，优先全骰覆盖、额外投掷、计数触发和稳定输出。
- 恶霸：偏 `LARGE`、`MEDIUM`，优先 4 格大件、3 格转大件、大件触发联动。
- 狗皇帝：偏 `GENERAL`、`BIG_DICE` 或围绕幸运数字的点数覆盖，优先 lucky 职业装备和能提高幸运数字收益的装备。

## 回合模拟

构筑器从 0 回合开始模拟到目标 `round`。每轮流程：

1. 增加简化预算：使用当前游戏收入节奏的近似值，保证高回合离线狗有足够资源。
2. 选择商店类型：按狗狗策略档案和回合阶段决定，不使用完全随机。
3. 生成候选装备：复用 `shopPool(shopType)` 获取合法普通装备。
4. 评分候选装备：按标签、点数覆盖、尺寸、与狗狗特性的联动、当前装备缺口和可升级性打分。
5. 买入或升级：优先购买能升级现有核心装备的副本，其次购买评分最高且可放入装备栏的装备。
6. 整理装备栏：优先把核心装备放入 `EQUIPMENT`，低价值 starter 或溢出装备留在背包外或丢弃。
7. 职业奖励：目标回合达到 3 或 6 时，从该狗的职业奖励中选择最符合策略的一个加入构筑。
8. 遗物选择：目标回合达到 4 后，根据策略和当前构筑选择 0 到 2 个遗物。

模拟不创建真实 `Run`，所有状态只存在于内存中。

## 装备选择与升级

离线狗应主动替换弱装备：

- starter 只保留能补足核心点数或短期触发密度的部分。
- 普通装备、职业装备和遗物都按狗狗策略评分。
- 两件相同 `defId` 且相同品质的装备会合成为更高品质。
- 品质最高到 `DIAMOND`，沿用 `nextQuality()` 和 `canUpgradePair()` 的语义。
- 装备栏空间不足时，优先保留高评分、高品质、职业联动更强的装备。

装备摆放仍使用现有格子规则：

- 使用 `findSlot()` 或等价规则寻找可放位置。
- 只输出 `area: "EQUIPMENT"` 的核心构筑装备。
- 输出顺序按从左到右触发，优先把关键触发装备放在前面，联动装备靠近需要被影响的目标。

## 遗物选择

遗物只在 `round >= 4` 时出现。选择规则：

- 小点构筑优先 `midas-right` 或 `half-die-right`。
- 大点构筑优先 `midas-left` 或 `half-die-left`。
- 如果遗物会严重削弱当前装备触发覆盖，则降低优先级。
- 最多生成 2 个遗物，避免离线对手过早超过真实玩家平均强度。

## 难度与强度控制

构筑强度应随回合增长，但不能在低回合压倒玩家：

- `round 0-2`：保留较多 starter，只加入少量普通装备，最多银色品质。
- `round 3-5`：至少出现一个职业装备，普通核心装备可升到银或金。
- `round 6+`：允许第 6 回合职业装备、遗物组合和金色核心装备，少量高回合可出现钻石。
- `wins` 提高会略微增加预算和升级倾向；`losses` 提高会略微降低构筑质量。

这些规则让同一回合的离线狗不会完全相同，也能匹配玩家当前胜负区间。

## 确定性

构筑器使用 `createRng(seed)`。种子由 `round`、`wins`、`losses` 和狗狗类型组成。所有随机选择都必须走该 RNG，不使用 `Date.now()` 或 `Math.random()`。

确定性要求：

- 同一输入生成完全相同的 `dogType`、`luckyNumber`、装备、品质、位置和遗物。
- 不同回合或胜负记录会产生可预测但有差异的构筑。

## 测试

使用 TDD 添加服务器测试：

- `seedGhost()` 在高回合会生成职业装备和非 starter 装备，不再只是固定四件升级。
- 不同狗狗生成的装备标签和职业装备有明显差异。
- 离线狗会升级重复装备品质。
- `round >= 4` 的构筑可以携带遗物。
- 相同输入结果确定性一致。
- 输出装备都能通过现有格子规则校验，不重叠、不越界。
- API fallback 匹配仍能返回 `matchedGhost`，且战斗可以正常开始。

## 风险

- 如果评分规则过强，低回合离线狗可能变得不公平。用回合段限制品质和遗物数量控制。
- 如果评分规则过复杂，后续新增装备难维护。策略档案只描述偏好，具体分数保持简单、可读。
- 当前部分中文字符串在源码中显示为乱码，本次设计不修复编码问题，避免扩大范围。

## 验收标准

- 没有真实 ghost 时，匹配到的离线狗不再是固定装备序列。
- 第 3 回合后的离线狗可能携带符合狗狗类型的职业装备。
- 第 4 回合后的离线狗可能携带遗物。
- 离线狗会根据犬种偏好形成小点、大点、额外投掷、大件或幸运数字构筑。
- 所有新增和既有相关测试通过。
