# 忍法·破额外触发设计

## 背景

`忍法·破` 当前效果为：装备不再只按绑定点数触发，也会在骰点等于装备容量时触发。玩家要求在这个基础上增强效果：按照容量触发时，有 50% 概率额外触发 1 次。

## 目标

- 保留现有“按容量触发”的判定。
- 仅当装备因为“容量等于骰点”命中时，才进行 50% 额外触发判定。
- 额外触发必须是真正的第二次装备触发，而不是把数值直接乘以 1.5。
- 更新道具描述、测试和外部数值文档。
- 修改后运行 `npm run build`，确保 `dist-click\DogFight-standalone.cmd` 是最新可玩版本。

## 非目标

- 不改变普通点数命中的触发概率。
- 不改变点金手、半截骰、天命数字等已有匹配规则。
- 不调整装备基础伤害、治疗、护盾、价格或品质倍率。

## 行为设计

战斗结算生成初始触发队列时，先沿用现有 `matchingContext()` 判断每件装备是否命中。命中后如果上下文表明该装备是通过 `TRIGGER_BY_SIZE` 的容量规则命中，则额外消耗一次随机数做 50% 判定；成功时把同一件装备再加入一次初始队列。

这个额外入队会让装备完整执行第二次，包括伤害、治疗、护盾、状态、相邻触发、失效消耗和现有触发队列上限。它不直接修改单次效果数值，因此与现有联动规则更一致。

## 测试策略

- 新增一个失败测试：装备原本不匹配骰点，但因容量命中；在固定随机种子下 50% 判定成功时，同一装备在同一次投掷中触发两次。
- 验证描述更新：`shiba-break` 的品质描述包含“50% 概率额外触发 1 次”。
- 跑定向测试后跑完整构建。
