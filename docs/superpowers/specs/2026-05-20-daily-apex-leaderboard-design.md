# 当日巅峰榜设计

## 目标

在现有巅峰榜之外增加“当日巅峰榜”。玩家投入一只完成局后，同一只狗会同时挑战总榜和当日榜；前端通过页签在两个榜单之间切换。

## 时间口径

当日榜按中国时区 `Asia/Shanghai` 的自然日统计，但刷新点为每日 05:00。

- `2026-05-20 05:00:00` 到 `2026-05-21 04:59:59` 归入 `2026-05-20` 当日榜。
- `2026-05-20 04:59:59` 仍归入 `2026-05-19` 当日榜。

## 数据模型

复用 `ApexEntry`，新增：

- `boardType`：`OVERALL` 或 `DAILY`。
- `boardKey`：总榜固定为 `default`，当日榜为 `YYYY-MM-DD`。

唯一约束从全局 `sourceRunId` 和全局 `rank` 改为：

- `sourceRunId + boardType`：同一只完成局在总榜只能一次、在当日榜只能一次。
- `boardType + boardKey + rank`：每个榜单内部排名唯一。

## 后端流程

`GET /api/apex` 返回：

- `leaderboards.overall`
- `leaderboards.daily`
- `dailyBoardKey`
- `dailyResetHour: 5`
- `candidates`

`POST /api/apex/submit`：

1. 校验跑局属于当前用户且已完成。
2. 校验该 `sourceRunId` 没有进入过总榜，防止重复投入。
3. 确保总榜和当前当日榜各有 50 个种子条目。
4. 用同一份狗快照分别挑战总榜、当日榜。
5. 在同一个事务里分别插入两条 `ApexEntry`，并维护两个榜单的独立排名链。
6. 返回两个榜单、总榜报告和当日榜报告。

## 前端交互

巅峰竞技场右侧榜单增加页签：

- `总榜`
- `当日榜`

默认显示总榜。当日榜标题提示“每日 05:00 更新”。提交后结果卡片展示总榜名次和当日榜名次。

## 验证

- 后端 API 测试覆盖：一次提交会同时写入总榜和当日榜；重复提交仍返回 409。
- 纯逻辑测试覆盖：`Asia/Shanghai` 05:00 前后生成不同当日榜 key。
- 前端结构测试覆盖：存在总榜/当日榜页签和当日榜 05:00 文案。
- 修改会影响前端、后端和可玩单文件版本，交付前必须运行 `npm run build`。
