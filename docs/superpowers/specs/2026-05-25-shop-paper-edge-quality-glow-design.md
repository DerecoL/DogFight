# 商店装备纸条边缘与品质柔光设计

## 背景

当前商店装备卡片已经使用不规则 `clip-path` 模拟纸条边缘，但品质表现仍然依赖较粗的品质色 `border`。浏览器原生边框不会沿着 `clip-path` 的波动均匀变化，导致边缘看起来像硬质矩形框，和纸张质感冲突。

## 目标

- 每个装备卡片优先表现为一张小纸条，而不是一个带粗品质边框的盒子。
- 纸条边缘使用细的深咖色描边，描边要能从背景中清楚分离出来，但不能厚重。
- 描边视觉上要跟随纸条不规则边缘，避免直线粗框。
- 品质不再靠粗色边框表达，改为纸条边缘向外发出的品质色柔光。
- 柔光比视觉稿 B 更明显，但不能压过纸张本体，也不能变回粗色框。

## 范围

- 重点修改商店商品卡片：`.paper-shop-card`、`.shop-card` 及其品质类。
- 同步检查装备卡片通用品质边框：`.item-card`、`.paper-item-card`，避免仍出现明显粗品质框。
- 不修改装备数值、战斗逻辑、商品数据、价格、品质倍率或交互流程。
- 不修改外部数值表。

## 视觉方案

采用“细深咖色纸边 + 更明显品质柔光”：

- 纸条本体继续使用米白纸张、纸纹和轻微不规则切边。
- 元素本身不再使用 4px/5px 的品质色边框。
- 纸边描边使用深咖色，约 1px 到 1.5px 的视觉厚度，颜色比背景更深，保证能分辨纸条轮廓。
- 描边通过跟随 `clip-path` 的层或内阴影实现，而不是依赖普通粗 `border`。
- 品质色以外发光表达：
  - 青铜：暖棕/琥珀柔光，低强度。
  - 白银：冷白/浅蓝灰柔光，中低强度。
  - 黄金：金色柔光，中高强度。
  - 钻石：青蓝柔光，高强度但保持柔边。
- 选中、悬浮、已有、不可购买状态不得覆盖纸边识别；这些状态只能叠加辅助阴影或轻微轮廓。

## 实现约束

- 保留当前纸条的 `clip-path` 波动和每个商品的轻微倾斜。
- 使用 CSS 变量定义纸边、纸张阴影、品质柔光，便于统一调节。
- 如果需要伪元素，应避免遮挡卡片内容和点击。
- `overflow`、`clip-path`、`box-shadow` 的组合要保证柔光可见，同时纸条边缘不出现硬矩形外框。
- 移动端布局不能因为描边或光效导致文字溢出或卡片尺寸变化。
