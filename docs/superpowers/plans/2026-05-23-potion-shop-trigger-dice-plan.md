# 药水商店触发点数实现计划

## 目标

新增“药水商店”，让玩家通过三选一药水修改一件非职业装备实例的基础触发骰子值。药水修改的是装备实例的基础触发点数，战斗和展示会先读取这个覆盖值，再继续套用遗物、附魔和其他道具规则。

## 核心规则

- 药水商店是三选一奖励商店。
- 药水出现概率与遗物、升级相同，均为第 4 回合后独立 33% 判定。
- 每个药水选项独立按权重 `50,10,10,10,1` 先抽类别，再抽具体点数或范围。
- 类别 1：增加 1 个随机点数，点数为 1/2/3/4/5/6，已有点数不会产生额外触发。
- 类别 2：增加 2 个随机唯一点数，点数为 1/2/3/4/5/6，已有点数不会产生额外触发。
- 类别 3：额外增加 1 个随机点数，用于强化某件指定装备，结果仍按触发点数集合去重。
- 类别 4：将触发修改为小点、大点或极值之一。
- 类别 5：修改为全点数触发，是最低概率稀有结果。
- 职业装备不可使用药水。

## 实现范围

1. 新增药水模型模块 `src/server/game/potion.ts`，负责权重随机、文案生成、点数规范化和基础骰子修改。
2. 为跑局数据增加 `potionChoices`，为装备实例增加 `triggerDiceOverride`。
3. 新增 Prisma migration，把药水选择和实例触发点覆盖持久化。
4. 将 `POTION` 接入商店三选一入口，将 `POTION_CHOICE` 接入跑局阶段流转。
5. 新增 `/api/runs/:runId/potion/select`，校验药水、目标装备和职业装备限制。
6. 战斗触发和 UI 触发点展示统一读取 `triggerDiceOverride ?? def.dice` 作为基础触发点。
7. 前端新增药水三选一界面，玩家先选择药水，再点选背包或装备区中的目标装备。
8. 狗狗乱斗自动流程处理 `POTION_CHOICE`，自动把第一瓶药水应用到第一件非职业装备。
9. 同步外部 Excel 数值文档，补充药水规则、触发点模型、商店概率和职业装备限制。
10. 跑测试、构建和浏览器烟测，确认单文件可玩版本已刷新。
