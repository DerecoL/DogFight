# DogFight 表现特效优化与下一步整理

## 目标

在不切换现有柔和手绘风格的前提下，把后续优化重点从“界面材质精修”推进到“表现特效与因果反馈”。核心目标是让玩家清楚感知：谁触发了什么、效果飞向哪里、命中了谁、造成了什么结果，以及操作成功或失败的原因。

## 当前已完成的表现基础

当前几轮已经完成了以下基础能力：

- 全局手绘视觉系统：纸张纹理、桌面洗色、粗描边、轻微旋转、贴纸、柔和阴影。
- 高频 UI 材质：大厅、狗狗选择、商店、背包、遗物栏、奖励选择、浮动提示、战绩面板。
- 操作反馈：按钮按压、高亮、hover 轻微旋转、拖拽物纸片感、格子放置反馈、出售区反馈、升级角标。
- 战斗基础表现：手绘战斗舞台、骰子弹跳、状态芯片、结果卡翻出、战斗日志纸条化。
- 战绩与弹层细节：战绩详情背景纸纹、选中洗色、空状态内框、日志色点。

这些内容已经把画面从“方正网页 UI”推进到了“手绘桌游界面”，但核心战斗特效仍然偏弱，主要缺少连续的动作因果链。

## 当前表现短板

### 1. 战斗事件缺少视觉因果链

现在玩家能看到骰子、日志、状态和结果，但还不够明确地看到：

- 哪个装备被触发。
- 特效从哪个装备或角色发出。
- 特效飞向哪个目标。
- 命中后是伤害、治疗、护盾、毒、虚弱、冻结还是反伤。

下一步需要让“装备触发 -> 轨迹 -> 命中 -> 数字/状态变化”成为战斗表现的核心。

### 2. 数字反馈不够有戏剧性

伤害、治疗、护盾、中毒、虚弱等结果需要做成手写弹字，而不是只依赖日志和血条变化。数字应有颜色、运动方向和层级差异：

- 伤害：红色手写数字，短促上弹，目标轻微受击。
- 治疗：绿色数字上浮，带柔和圆环。
- 护盾：蓝色数字或盾牌碎片，落在血条或角色前方。
- 中毒：草绿色数字，带轻微雾化。
- 虚弱：紫色“减弱”贴纸或下坠波纹。
- 冻结：冰蓝霜边和短暂停顿感。

### 3. 状态表现还可以更贴近角色身体

现在状态芯片已经有手绘化，但角色本体上的状态覆盖还可以增强：

- 护盾：角色前方透明蓝色泡泡或手绘盾线。
- 中毒：角色周围浅绿色毒雾和小点。
- 虚弱：紫色波纹或下垂线条。
- 冻结：角色边缘霜线，不做硬科幻冰块。
- 荆棘：金色刺点或短线闪光。

### 4. 商店与背包的完成反馈还不够完整

商店、背包、升级、出售已经有基础手绘感，但缺少“动作完成”的小高潮：

- 购买成功：商品短暂弹起并飞向背包方向，价格牌或金币轻跳。
- 金币不足：价格牌变红、按钮轻抖、浮出一张错误小贴纸。
- 升级成功：两个物品靠拢、星星闪出、新品质边框出现。
- 出售成功：出现小收据或金币纸片。
- 放置成功：格子轻弹并出现柔和描边。
- 放置失败：红色草稿线或小叉号，不打断操作节奏。

### 5. 奖励和遗物获得缺少仪式感

奖励选择现在已经更像纸卡，但获得瞬间还可以加强：

- 职业奖励：卡片翻开、盖章、星星纸屑。
- 遗物奖励：三张特殊徽章卡从桌面抽出。
- 选择后：选中遗物贴到遗物栏，未选项淡出或收回。

## 下一步优化优先级

### 第一优先级：战斗特效因果链

这是最能提升游戏高级感的部分。

建议先做一个轻量事件表现层，不改战斗规则，只消费现有 battle playback 事件：

1. 给每个战斗事件生成表现类型。
2. 在 `BattleFxCanvas` 中绘制短生命周期特效。
3. 给装备卡、目标狗、骰子和日志增加同步高亮。
4. 保持所有特效在 reduced motion 下可关闭或降级。

验收标准：

- 玩家能看出当前是谁在行动。
- 玩家能看出哪个装备被触发。
- 玩家能看出效果打到谁。
- 玩家能看出结果类型。
- 不影响战斗播放速度和移动端布局。

### 第二优先级：手写弹字与状态覆盖

在战斗因果链之后补数字和状态层。

建议新增一组视觉类型：

- `damage`
- `heal`
- `shield`
- `poison`
- `weak`
- `freeze`
- `thorns`
- `miss`

每种类型对应颜色、轨迹、停留时间和目标挂点。实现上优先复用 canvas，避免在 React 中频繁创建大量 DOM 节点。

验收标准：

- 数字可读，不遮挡核心血条太久。
- 同一时间多段伤害不会糊成一团。
- 状态覆盖轻量，不把狗狗图像盖死。
- reduced motion 下保留颜色和静态提示。

### 第三优先级：商店、背包、升级、出售的完成反馈

这部分提升操作爽感。

建议按照频率排序：

1. 购买成功与金币不足。
2. 放置成功与非法放置。
3. 升级成功。
4. 出售成功。
5. 刷新商店纸牌替换。

验收标准：

- 操作反馈在 300-700ms 内完成。
- 不改变拖拽命中区域。
- 不造成布局跳动。
- 动画失败或被跳过时，游戏状态仍清晰。

### 第四优先级：奖励与遗物仪式感

这部分用于提升阶段性成就感。

建议做：

- 职业奖励翻卡和盖章。
- 遗物三选一徽章卡。
- 选中后贴入遗物栏的短动画。
- 未选项淡出或纸片收回。

验收标准：

- 奖励页仍然清楚易选。
- 动画不拖慢流程。
- 移动端不遮挡确认按钮。

## 技术约束

- 不切换美术风格。
- 不改战斗规则、装备数值、遗物数值和后端接口。
- 优先在 `src/App.tsx` 和 `src/App.css` 内增量实现。
- 高频区域不改变点击、拖拽和放置命中范围。
- 所有动画遵守 `prefers-reduced-motion`。
- 每次影响前端展示或可玩版本后必须运行 `npm run build`。
- 构建成功后确认 `E:\AI-GPT\DogFight\dist-click\DogFight-standalone.cmd` 已重新生成。

4. 给目标狗增加轻量受击/治疗/护盾反馈。
