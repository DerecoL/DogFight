## 用户界面与交付偏好

- 最高优先级交接规则：用户需要阅读或审阅的每一份规格说明或 Markdown 文档，都必须同时提供可点击的聊天链接和明确的绝对存储路径。
- 告诉用户本地文件位置时，必须用纯文本或代码块明确给出原始绝对文件系统路径。
- 不要只依赖可点击 Markdown 文件链接，因为用户的应用里可能无法正确打开。
- 对 `.env`、配置文件、日志、生成产物等可操作本地文件，优先明确展示直接绝对路径。
- 交付文档时，始终同时提供：
  1. 可在对话中直接打开的 Markdown 文件链接
  2. 标注为存储路径的原始绝对文件系统路径
- 在 Windows 上，可点击本地文件链接要在盘符前加 `/`，例如：`[name](/C:/Users/User/path/to/file.md)`。
- 交付文档时，不能只给纯文本路径。
- 本地文档交付必须使用以下两行格式：
  `[file-name](/absolute/path/to/file.ext)`
  `存储路径：absolute-path`
- 这个要求对每一次 `.md` 审阅交付和每一份规格文档都强制生效。
- 与规则、逻辑、规格、工作流或行为说明相关的 Markdown 文件，默认用中文编写，方便用户直接审阅。

## Python 版本

- 在 Codex 中，默认所有 Python 引用和命令都使用 `python3`，除非用户明确要求其他解释器。
- 最高优先级 Python 版本规则：默认始终调用并使用 Python 3.x 或更新版本，除非用户明确要求，否则绝不回退到 Python 2.7。

## 图片加载性能规则

- 后续工程中凡是新增、修改或处理图片相关内容，都必须优先关注加载速度，不能只关注视觉效果。
- 引入图片资源前，应优先考虑文件体积、尺寸、格式、压缩、缓存、懒加载、预加载时机和首屏影响。
- 对用于前端展示的图片，默认避免无必要的大尺寸原图直出；应按实际显示尺寸提供合适资源，必要时使用 WebP、AVIF 或其他更适合的压缩格式。
- 对首屏、战斗界面、资源列表、图标、头像、背景图等会影响用户感知速度的图片改动，交付前必须检查是否造成明显加载变慢、闪烁、布局抖动或阻塞交互。
- 如果为了画质、兼容性或实现成本选择了更大的图片资源，最终回复中要说明取舍原因。

## 新增装备/遗物资源规则

- 后续新增任何装备或遗物时，不能只新增数据、文案或战斗逻辑；必须同步进入 icon 制作流程。
- 每个新增装备或遗物的正式展示 icon 必须使用当前统一的“透明底手绘贴纸风格”WebP：`public/assets/sticker-icons/<id>.webp`。
- 贴纸 icon 必须是 128x128 WebP、透明背景、主体完整、白色贴纸描边、粗轮廓、无文字、无卡片边框、无圆角底框、无残留切图边线，并保证在 30px、52px、74px 三档可读。
- 新增装备或遗物不得使用临时占位图、简化线框图、自动生成的扁平 SVG 截图、复用不匹配语义的旧 icon，除非用户明确接受临时占位并在最终回复中说明。
- 新增图标必须接入前端静态映射，例如 `src/App.tsx` 的 `itemIcons` 或 `relicIcons`，不能依赖 fallback 图标作为正式交付。
- WebP 贴纸图标应优先控制体积；单个新增 WebP 默认不得超过 15KB，除非最终回复说明画质、兼容性或实现成本取舍。
- 新增装备或遗物交付前必须有资产测试，至少检查 WebP 存在、WebP 文件体积、前端映射、无旧 `items/relics` SVG 引用、无缺图 fallback。
- 新增或替换图标时，必须检查是否影响首屏、战斗界面、商店列表、装备卡和遗物栏的加载速度、闪烁、布局抖动或交互阻塞。

## 单文件可玩版本同步规则

- 每次完成会影响游戏行为、前端展示、资源、装备数据、遗物数据、随机离线狗数据、战斗逻辑或打包逻辑的代码修改后，必须运行 `npm run build`。
- `npm run build` 会同步重新生成可直接发给别人玩的单文件版本：`E:\AI-GPT\DogFight\dist-click\DogFight-standalone.cmd`。
- 交付前必须确认 `npm run build` 成功；如果构建失败，要先修复失败原因，不能把旧的 `dist-click\DogFight-standalone.cmd` 当作最新版本交付。
- 如果本次修改只涉及不会影响可玩版本的纯文档内容，可以不用运行 `npm run build`，但最终回复里要明确说明没有运行的原因。

## 数值文档同步规则

- 每次完成会影响装备数值、遗物数值、职业道具数值、战斗参数、触发点数、品质倍率、治疗、护盾、伤害、毒、虚弱、控制、经济价格或任何平衡模型的代码修改后，必须同步更新外部数值文档：`C:\Users\User\Desktop\狗骰乱斗\狗骰乱斗.xlsx`。
- 更新数值文档时，必须同步检查并修改相关的基础数据页、模型页和看板页，例如 `通用装备`、`职业道具库`、`数值模型-装备期望`、`数值模型-平衡看板`，确保源码数值、描述文本、期望收益和统计摘要一致。
- 如果本次修改只涉及不会影响数值或平衡模型的纯 UI、文案、构建、部署或文档内容，可以不用更新该 Excel，但最终回复里要明确说明没有更新的原因。
